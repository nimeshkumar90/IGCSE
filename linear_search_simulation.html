Section A — Multiple Choice
Question 1 · Multiple Choice
In Linear Search, what is the worst-case number of comparisons for a list of n elements?
✅ C — n comparisons. In the worst case the target is the last element, or not in the list at all. Every element must be compared, giving O(n) comparisons.
Question 2 · Multiple Choice
What is the purpose of the Found ← FALSE flag at the start of the algorithm?
✅ B. The Found flag is a Boolean variable initialised to FALSE. When the target is matched, Found ← TRUE. This also helps the UNTIL condition stop the loop early.
Question 3 · Multiple Choice
The loop in this Linear Search uses UNTIL Found OR Counter > ClassSize. What does this condition guarantee?
✅ B. The OR condition provides two exit routes: early exit when a match is found (Found = TRUE), or natural termination after every element has been checked (Counter > ClassSize).
Question 4 · Multiple Choice
Does Linear Search require the list to be sorted before searching?
✅ C — No sorting required. This is a key advantage over Binary Search. Linear Search checks every element in sequence and works on any unsorted or sorted list.
Question 5 · Multiple Choice
In the best case, Linear Search finds the target after how many comparisons?
✅ B — 1 comparison. The best case is when the target is the very first element in the list. The condition matches immediately, Found ← TRUE, and the loop exits after a single check.

How many comparisons are made before the algorithm finds the name and sets Found ← TRUE?

Counter=1: "Alice" = "Charlie"? → No → Counter ← 2
