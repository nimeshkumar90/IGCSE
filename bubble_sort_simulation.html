📝 Bubble Sort Assignment

Test your understanding with these questions on Bubble Sort concepts.

Section A — Multiple Choice
Question 1 · Multiple Choice
How many passes does Bubble Sort need to guarantee sorting an array of n elements?
✅ C — n − 1 passes. After each pass, the largest unsorted element is placed in its final position. So after n − 1 passes, all elements are in their correct positions.
Question 2 · Multiple Choice
Given the array [3, 1, 4, 1, 5], after the first pass of Bubble Sort, what will the array look like?
✅ B — [1, 3, 1, 4, 5]. Pass 1: compare 3&1 → swap → [1,3,4,1,5]; compare 3&4 → no swap; compare 4&1 → swap → [1,3,1,4,5]; compare 4&5 → no swap. Final: [1,3,1,4,5].
Question 3 · Multiple Choice
What is the worst-case number of comparisons for Bubble Sort on an array of n elements?
✅ C — n(n−1)/2. In the worst case (reverse-sorted array), Pass 1 makes n−1 comparisons, Pass 2 makes n−2, … totalling n(n−1)/2 ≈ O(n²).
Question 4 · Multiple Choice
In the optimised Bubble Sort, the algorithm stops early when:
✅ B — No swaps occur in a complete pass. If a full pass completes with no swaps, the array is already sorted and further passes are unnecessary. The Swap ← FALSE flag tracks this.
Question 5 · Multiple Choice
After each pass of Bubble Sort, which element is guaranteed to be in its final correct position?
✅ C — The largest remaining unsorted element. Each pass "bubbles up" the largest value to the end of the unsorted portion, hence the name Bubble Sort.
Section B — Fill in the Blanks
Question 6 · Fill in the Blank
For an array of 6 elements, the maximum number of swaps that can occur during the entire Bubble Sort is swaps.
✅ 15 swaps. Maximum swaps = n(n−1)/2 = 6×5/2 = 15 (occurs when array is fully reversed).
Question 7 · Fill in the Blank
In the pseudocode, the variable Swap is set to at the start of each pass.
✅ FALSE. Swap ← FALSE is set at the beginning of each pass. If any swap occurs during the pass it becomes TRUE, otherwise the loop ends early.
Question 8 · Fill in the Blank
The temporary variable used during a swap (e.g. Temp ← Temperature[Index]) ensures that the original value is not during the swap.
✅ Lost / Overwritten. Without a temp variable, assigning A ← B would destroy A's original value before it can be placed into B.
Section C — Trace Questions
Question 9 · Trace
Trace Bubble Sort on the array [9, 5, 2, 7].

How many total swaps occur across all passes?

✅ 4 swaps.
Pass 1: [9,5,2,7] → swap 9&5 → [5,9,2,7] → swap 9&2 → [5,2,9,7] → swap 9&7 → [5,2,7,9] (3 swaps)
Pass 2: [5,2,7,9] → swap 5&2 → [2,5,7,9] (1 swap)
Pass 3: [2,5,7,9] → no swaps → done
Total = 4 swaps.
Question 10 · Trace
The array [1, 2, 3, 4, 5] is already sorted.
In the optimised Bubble Sort, how many passes will be performed before the algorithm stops?

✅ 1 pass. The optimised version runs a single pass, finds no swaps (Swap stays FALSE), and terminates immediately. This is the best-case O(n) performance.
